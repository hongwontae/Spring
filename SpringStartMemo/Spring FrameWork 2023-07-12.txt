@Scope를 통해 프로토 타입을 주면 기존의 객체를 가져오는 싱글톤 방식이 아니라 새로운 객체를 만들어 내서 그것을 사용한다. 

Spring IOC 컨테이너 하나에는 객체 인스턴스가 하나이다. ioc 컨테이너가 두 개를 경우 객체 2개 싱글톤 아닐수도...
Java 싱글톤은 java 가상 머신 하나당 객체 인스턴스가 하나이다.

@PostConstruct, @PreDestroy

의존성을 연결하는 대로 초기화를 실행하려면? => @PostConstruct
=> 초기화를 수행하기 위해 의존성이 주입 완료된 후 실행해야 하는 메서드에서 사용하는 어노테이션
=> 이 메서드는 클래스를 사용하기 전에 호출되어야 한다. 
=> 다른 빈이 이 bean을 사용할 수 있게 되기 전에 메서드가 호출당할 것이다.

컨텍스트가 종료되기 전에 무엇가를 해야될 로직(메서드)를 위한 어노테이션?
=> preDestory
컨테이너에서 인스턴스를 삭제하는 과정 중에 있음을 알려주는 콜백 알림으로 메서드에서 사용되는 애노테이션
